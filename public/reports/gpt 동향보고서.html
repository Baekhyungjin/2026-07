Assistants API는 2026년 8월 26일 종료 예정이며 Responses API로 이전해야 합니다. 또한 gpt-5.2-chat-latest와 gpt-5.3-chat-latest 스냅샷은 8월 10일 종료 예정이므로 7월 중 마이그레이션 점검이 필요합니다.

주의
7월 “예정”은 확정 일정과 전망을 구분해야 합니다
GPT-5.6의 더 넓은 제공은 공식적으로 “coming weeks”라고만 공개되었으므로, 정확한 날짜를 단정하면 안 됩니다. 보고·강의에서는 공식 발표: 제한 프리뷰 시작 / 향후 몇 주 내 확대 예정으로 표현하는 것이 안전합니다.
목회AI연구소 강의용 해석
🧠
Memory = 개인화된 사역 비서
설교 스타일, 교회 상황, 선호 출력물, 반복되는 사역 맥락을 기억하게 하여 매번 처음부터 설명하지 않도록 설계합니다.
⏰
Scheduled Tasks = 반복 업무 담당자
주간 사역 브리핑, 강의 신청 현황 점검, 뉴스 모니터링, 일정 알림 같은 반복 업무를 예약합니다.
✍️
Writing Blocks = 문서 생산 라인
공지문, 칼럼, 보고서, 강의안, 제안서를 대화형 초안이 아니라 편집 가능한 문서 단위로 관리합니다.
🧑‍💻
Codex = 내부 앱·자동화 개발자
구글시트, Apps Script, 웹페이지, 신청폼 후처리, HTML 보고서 같은 실무 자동화를 담당하게 합니다.
🛡️
Security = 교회 데이터 보호 기준
Lockdown Mode, Active Sessions, 앱 권한 제어를 통해 개인정보·상담·헌금·출석 데이터 취급 기준을 설명합니다.
🎤
강의용 5문장 요약
바로 사용
1
GPT는 더 이상 질문에 답하는 도구만이 아닙니다.
기억하고, 정리하고, 예약하고, 반복 업무를 맡는 작업 환경으로 바뀌고 있습니다.
  • 오프닝 멘트로 사용하기 좋습니다.
2
6월 업데이트의 핵심은 메모리와 예약 작업입니다.
나를 아는 AI, 내가 맡긴 일을 잊지 않는 AI로 발전하고 있습니다.
  • Memory와 Scheduled Tasks를 연결해 설명하면 이해가 빠릅니다.
3
Codex는 개발자만의 도구가 아니라 업무 자동화의 핵심입니다.
교회 행정, 보고서, 신청폼, 콘텐츠 제작 자동화에도 연결됩니다.
  • 비개발자에게는 “내부 앱을 만들어주는 직원”으로 설명하세요.
4
7월에는 GPT-5.6 확대 여부를 주목해야 합니다.
Sol, Terra, Luna가 실제 사용자 환경에 어떻게 제공되는지가 관전 포인트입니다.
  • 확정 날짜가 아니라 공식 표현 “향후 몇 주”를 유지하세요.
5
AI를 잘 쓰는 사람은 기능을 많이 아는 사람이 아니라 흐름을 설계하는 사람입니다.
메모리, 예약, 문서, 이메일, Codex, 보안을 하나의 운영 체계로 묶어야 합니다.
  • 마무리 메시지로 적합합니다.
